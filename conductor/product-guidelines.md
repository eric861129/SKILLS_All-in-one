# Product Guidelines

## 1. 視覺設計 (Visual Design)
- **核心風格：** 採用 **極簡與清爽 (Minimalist)** 與 **反 AI Slop 高階感 (Premium)** 的設計語彙。
- **字體規範：** 標題使用 `Outfit` 展現現代感，正文與代碼採用 `JetBrains Mono` 或 `Inter` 確保工程專業性與可讀性。
- **排版層級：** 嚴格遵循字體層級，優先使用空白與精確的邊框（Slate-800/50）來引導使用者視覺，而非依賴過度修飾。
- **元素規範：** 嚴禁在 UI 中使用 Emoji 作為功能圖標。採用磨砂玻璃 (Glassmorphism) 效果增添層次感，並避免使用預設漸層色。

## 2. 互動體驗 (Interaction Experience)
- **感官反饋：** 互動應表現出 **輕快與即時 (Snappy)** 的特質。
- **性能優先：** 確保狀態切換與頁面跳轉零延遲，所有動畫應短促且有力，提供高效的工具感。

## 3. 文案語調 (Tone of Voice)
- **溝通風格：** 採用 **專業與簡潔 (Professional)** 的語調。
- **內容原則：** UI 文案應直接了當，移除冗餘的修飾語，專注於提供工程層面的清晰指令與技術細節。

## 4. UX 原則 (UX Principles)
- **反 Slop 設計：** 拒絕通用、廉價的 AI 生成美學，致力於提供符合高階工程師直覺的質感工具。
- **一致性：** 確保所有組件與頁面在視覺與互動行為上保持高度一致。
